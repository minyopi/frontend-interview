# Recoil을 사용한 이유?

Recoil은 보다 단순하게, 그리고 기존 상태 관리자들만으로 해결하기 어려웠던 다음의 문제들을 해결하기 위해 개발되었다.

- 유연하게 공유 가능한 상탯값(Flexible shared state)
- 파생 데이터 활용 및 쿼리 기능(Derived data and queries)
- 앱의 전반적 상태 관찰(App-wide state observation)
